# Phase 3 UI 스펙

## 1) 기능 우선 원칙
- 디자인 리뉴얼보다 운영 기능 완성도를 우선한다.
- 기존 컴포넌트 스타일을 유지하고, 정보 밀도/가독성 중심으로만 개선한다.

## 2) 화면 구성
## `/schedules`
- 스케줄 목록(활성/다음실행/최근실행/상태)
- 스케줄 생성/수정 폼
- 즉시 실행/중지/재개 액션

## `/policies`
- 정책 목록(활성/범위/기준/최근 액션)
- 정책 생성/수정
- 정책 활성/비활성 토글
- 정책 액션 이력 테이블

## `/agents`
- 에이전트 상태 카드(online/degraded/offline)
- heartbeat 시간, capacity, queue 할당 표시
- 노드 장애 감지 배지

## 3) 공통 UX
- 상태값은 배지로 통일
- 위험 액션(일시중지/비활성)에는 확인 단계 제공
- URL query 기반 필터 유지
