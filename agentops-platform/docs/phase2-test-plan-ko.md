# Phase 2 테스트 계획

## 1. 목표
- 템플릿 레지스트리/버전 비교/고급 검색의 핵심 회귀를 자동 검증한다.

## 2. 테스트 범위
- API 단위/통합 테스트
- 대시보드 핵심 사용자 흐름 테스트
- 성능/쿼리 제한 기본 점검

## 3. 시나리오
## A. 레지스트리
1. 템플릿 생성 성공
2. 동일 템플릿명 중복 생성 실패(409)
3. 버전 추가 성공
4. 기본버전 전환 성공
5. 비활성 템플릿/버전 사용 차단

## B. 작업 생성/실행
1. default 버전 resolve 후 실행 성공
2. 특정 버전 지정 실행 성공
3. 존재하지 않는 버전 요청 실패(400)
4. run 메타에 `template_version/adapter_version` 저장 확인

## C. 검색
1. 기간 필터 정확성
2. 상태 필터 정확성
3. 템플릿/버전 필터 정확성
4. 오류 키워드 검색 정확성
5. 페이지네이션(limit/offset) 정상 동작

## D. 버전 비교
1. 버전 2개 이상 비교 결과 반환
2. 지표 계산 정확성(success/failure/avg/p95)
3. 실행건 0인 버전 처리(분모 0)

## 4. 자동화 방식
- 스크립트: `scripts/e2e_phase2.sh` (신규)
- 실행 타겟: `make test-e2e-phase2` (신규)
- 기존 `make test-e2e`와 분리 운영

## 5. 합격 기준
- 필수 시나리오 전부 PASS
- 테스트 실행 시간: 로컬 기준 5분 이내
- 실패 시 원인 로그를 콘솔에 즉시 출력

## 6. 비기능 점검
- 검색 API 응답시간(1000건 기준) 1초 내 목표
- 인덱스 미적용 쿼리 탐지(실행계획 점검)
