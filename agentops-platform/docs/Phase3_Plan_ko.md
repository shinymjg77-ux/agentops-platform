# 3단계 계획(Phase 3): 정책형 스케줄러 + 멀티머신 확장

## 1. 단계 목표
- 스케줄 기반 자동 실행을 안정적으로 운영한다.
- 정책 조건(실패율/지연/야간창 등)에 따라 자동 제어를 수행한다.
- 워커를 2대 이상으로 확장해 분산 실행 및 장애 내성을 확보한다.

## 2. 완료 기준(Definition of Done)
- 스케줄 CRUD/API/UI가 동작하고 예약 실행 이력이 저장된다.
- 정책 엔진이 기준 위반 시 자동 액션(일시중지/알림 강화/우선순위 조정)을 실행한다.
- 워커 노드 heartbeat가 수집되고 다운 노드 감지가 가능하다.
- 멀티워커 환경에서 작업이 정상 분산 실행된다.
- Phase 3 E2E 테스트(스케줄/정책/노드장애)가 통과한다.

## 5. 권장 일정(6~8영업일)
- Day 1: DB 확장 + 스케줄 CRUD API
- Day 2: 스케줄 실행기 + 실행 이력 저장
- Day 3: 정책 엔진 코어
- Day 4: 에이전트 heartbeat + 멀티워커 라우팅
- Day 5: 대시보드 3개 화면
- Day 6: E2E/부하 기초 테스트
- Day 7~8: 안정화/운영 문서/인수
